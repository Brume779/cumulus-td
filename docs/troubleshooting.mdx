---
title: Troubleshooting
description: Common symptoms and practical fixes for Cumulus setup, clustering, tracking, and output issues.
---

Use this page when the component is running but the result looks wrong. Start with source alignment and masks before changing cluster ID or smoothing controls.

## Source And Scene Problems

| Symptom | Likely cause | First fixes |
| --- | --- | --- |
| People appear in the wrong place | Source transform or global transform is off | Tune source Translation, Rotation, Scale, and Pivot before clustering. Confirm the floor and walls line up in the calibration view. |
| Cameras do not agree with each other | One source is offset, delayed, or scaled differently | Disable all but one source, align it, then re-enable sources one at a time. |
| Background becomes a cluster | Scene crop or masks still include fixed geometry | Tighten Max Bounds, add an inset include mask, then add exclude masks for furniture or walls. |
| Valid body points are missing | Crop, mask, binning, or cleanup is too aggressive | Use View Outside and the noise invert view to see which stage removes the points. |
| Cook time is too high | Point cloud is too dense | Increase source Binning, lower Thinning, tighten the scene crop, or reduce unnecessary visual output. |

## Clustering Problems

| Symptom | Likely cause | First fixes |
| --- | --- | --- |
| Two people merge into one cluster | Cells are too large, masks are too loose, or bodies are too close | Lower Cell Size, lower Vertical Scale, tighten masks, or raise Min Cell Points slightly. |
| One person splits into multiple clusters | Cells are too small or the body cloud has gaps | Raise Cell Size, lower Min Cell Points, lower Min Points, or reduce cleanup strength. |
| Small objects become people | Thresholds are accepting small blobs | Raise Min Points and Min Size. |
| Large background blobs survive | Size filtering is too permissive | Lower Max Size and remove the object with a mask when possible. |
| Distant people disappear | Distant bodies have too few valid points | Lower Min Points and Min Cell Points carefully, then check that masks are not cutting off the far end of the scene. |

## Tracking And ID Problems

| Symptom | Likely cause | First fixes |
| --- | --- | --- |
| IDs swap between people | Matching is too permissive or centroids are noisy | Lower Match Radius, lower Max Velocity, increase Validation Time, or improve source alignment. |
| IDs drop during short occlusion | Ghost hold is too short | Raise Hold Time. |
| Old IDs linger after people leave | Ghost hold is too long | Lower Hold Time. |
| False clusters get IDs | Validation is too fast | Raise Validation Time and check Min Points/Min Size. |
| Real people appear too late | Validation is too slow | Lower Validation Time after clustering is stable. |

## Output Problems

| Symptom | Likely cause | First fixes |
| --- | --- | --- |
| `raw` slots are empty | No valid cluster candidates | Check `info/status`, point count, masks, Min Points, Min Size, and Max Size. |
| `_exit` pulses unexpectedly | Input became invalid or a tracked slot was retired | Check source validity, unsafe coordinates, and status codes in `info`. |
| Bounds jitter | Bounds are computed from noisy edge points | Tune clustering first, then enable Filter Bounds and raise Bounds Trim %. |
| Head position jumps | Top-region points are sparse or noisy | Lower Head Region %, lower Head Min Cutoff, or disable Detect Head if head channels are not needed. |
| Smooth output is still wrong | Filtering is hiding an upstream issue | Turn filtering off, fix alignment and clustering, then re-enable smoothing. |

## Status Codes

| Code | Meaning | What to check |
| --- | --- | --- |
| `0` | Success | The plugin has valid input and produced output. |
| `1` | Idle/reset | Reset is active or the plugin is idle. |
| `-1` | Missing input | The clustering CHOP has no input. |
| `-2` | Too few channels | Input must provide XYZ as channels 0, 1, and 2. |
| `-3` | Empty input | Input exists but contains no samples. |
| `-4` | Null channel data | A required input channel has no readable data. |
| `-5` | Unsafe coordinate | Input contains invalid or unsafe coordinate values. |
| `-6` | Invalid parameters | Check runtime-clamped clustering parameters for invalid values. |
