---
title: Practical Tuning Notes
description: Recommended tuning order and fixes for common source, clustering, tracking, and filtering problems.
---

Tune in this order:

1. Sources
2. Scene crop
3. Masks
4. Point-cloud cleanup
5. Clustering
6. Cluster ID stability
7. Smoothing

Filtering should be the last step. It can remove visible jitter, but it will not fix bad source alignment or unstable clustering.

## Source And Mask Setup

### Alignment

If clusters appear in the wrong place, fix source Translation, Rotation, Scale, and Pivot before touching clustering controls. Your scene and global source transforms should reflect your physical world.

Multiple cameras should agree on the same floor, wall, and body positions before their clouds are merged.

### Background Cleanup

If extra background objects become clusters:

1. Make sure Max Bounds describes the global room/tracking volume from `0, 0, 0`.
2. Use the first local mask as a slightly smaller include box, usually about 10-30 cm inside the global scene box.
3. Add more local mask blocks for furniture or other fixed geometry.
4. Use View Outside to confirm whether unwanted points are outside the scene box or still inside the active volume.

### Source Isolation

If one camera is noisy or offset, disable that source with Active and tune with a single source first. Re-enable sources one at a time so you can see which camera creates the problem.

### Point Density

| Symptom | Try |
| --- | --- |
| Point cloud is too dense or expensive | Increase Binning or lower Thinning. |
| People become sparse or lose limbs | Lower Binning or raise Thinning before lowering Min Points. |

## Clustering

| Symptom | Try | Notes |
| --- | --- | --- |
| People merge into one cluster | Lower Cell Size, lower Vertical Scale, raise Min Cell Points slightly, tighten masks, or reduce the scene box. | Lower Cell Size improves precision when the point cloud is clean enough. |
| One person fragments into multiple clusters | Raise Cell Size, lower Min Cell Points, or lower Min Points. | Fragmentation usually means the point cloud has gaps, the voxel size is too small, or cleanup is removing valid body points. |
| Small objects become clusters | Raise Min Points and Min Size. | Use masks too if the object is fixed in the room. |
| Large wall or floor blobs survive | Lower Max Size and tighten the scene crop. | Broad geometry is usually better removed before clustering. |
| Clusters disappear at distance | Lower Min Points and Min Cell Points carefully. | Distant people usually have fewer valid points. |
| Vertical body sections split apart | Lower Vertical Scale. | This can reduce vertical fragmentation. |
| Stacked or vertical structures merge too easily | Raise Vertical Scale. | This separates vertical structure more strongly. |

## Cluster ID Stability

| Symptom | Try |
| --- | --- |
| Cluster IDs swap | Lower Match Radius, lower Max Velocity, increase Validation Time, or improve source alignment. |
| Clusters vanish during short occlusions | Raise Hold Time. |
| Ghost clusters linger too long after people leave | Lower Hold Time. |
| False clusters get IDs too quickly | Raise Validation Time. |
| Real people feel delayed before appearing | Lower Validation Time. |
| Fast movement causes IDs to drop | Raise Match Radius or Max Velocity. |
| Clusters jump across the room to the wrong person | Lower Match Radius or Max Velocity. |

Use Reset Clusters after major changes to clustering, ID, or source parameters so old state does not affect what you are evaluating.

## 1 Euro Filtering Tuning

The 1 Euro filters use two main controls per output family:

| Control Type | How to tune it |
| --- | --- |
| Min Cutoff | Baseline smoothing. Lower values are smoother but add more lag. Higher values are more responsive but show more jitter. |
| Speed Coeff | Motion response. Higher values let fast movement pass through with less lag while still smoothing slow jitter. Lower values keep smoothing more constant. |

### Suggested Order

1. Start with only Filter Position enabled.
2. Lower Position Min Cutoff until standing jitter is acceptable.
3. Raise Position Speed Coeff until walking motion feels responsive.
4. Tune Filter Bounds separately from position.
5. Tune Filter Head last.

### Output-Specific Notes

| Output | Notes |
| --- | --- |
| Position | Tune this first. It is usually the most important smoothing target. |
| Bounds | Bounds often jitter more than centroids, so Bounds Min Cutoff can usually be lower than Position Min Cutoff. If boxes lag behind moving people, raise Bounds Speed Coeff. |
| Head | Head points are often sparse and noisy, so head smoothing may need stronger filtering than centroid smoothing. If the marker floats or jumps, lower Head Min Cutoff or lower Head Region %. If it lags behind fast motion, raise Head Speed Coeff. |
| Velocity | Use this only if downstream logic needs a smoother velocity signal. Aggressive velocity smoothing can make motion triggers feel late. |

If output is smooth but wrong, turn filtering off and fix clustering first. Filtering should polish stable clusters, not hide split, merged, or misaligned clusters.
